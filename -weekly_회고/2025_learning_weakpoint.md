# 문제 진단
- 허송세월? 
- 원인: 비효율적인 공부방식
- 결과: 이력 상의 공백;하지만 0은 아니다. 문제 해결에 대한 노력,과정,결과가 존재하기 때문.
- 성과: 나의 장점은 곧 "악마는 디테일에 있다"
    - 빠른 진행 속도와 진척도는 내 약점. 하지만 이와 트레이드-오프 격의 논리적 디테일과 그로인한 창의성이 나의 장점.

# 분석과 솔루션
- 나를 알고 적(목표)를 알면 백전백승
    1. 논리적 정합성이 충족되지 못하면 쉬이 넘어가지 못함 -> 세부사항(악마는 디테일에 있다)을 충족하지 못하면 만족하지 못하고 붙잡고 있는 성격
    2. 발산적 직관 -> 연상과 탐색 위주의 지식 습득 과정에서 흥미를 느끼는 성격
        => 원대한 목표로 시작(원거리에서 접근 시도) 
        => 세부목표가 방대;직렬화X 
        => 압축적인 큰 목표들을 세분화하지 못하고 심리적으로 압도되어 갈피를 잃음.
    3. deterministic하게 구축된 계획과 커리큘럼에선 내부의 모순을 찾게 되거나 샛길 탐색을 하게 되어 적응이 어려움.
    => 이런 성격을 보완할 큰 틀이 없었음.
- 어떤 틀을 구축해야 할 것인가?
    - 핵심: "세분화"에 gpt 적극 활용.
        1. **상세 로드맵 구축**에 생성형ai의 도움을 받기(feat.gpt고문)
            - n개월 안에 목표에 도달하는 로드맵, 주간 세부목표, 일간 루틴
            - 프로젝트 과제 주도 학습 시스템 로드맵 -> 핵심: **구체적인 프로젝트 주제와 가이드라인, 템플릿**에 대한 도움받기
                - cs
                    /실습: 챕터별 코드스니펫화
                    /최종장: 각 과목(OS/compiler/DB/...)에 해당하는 심화 프로젝트
                - math
                    /실습: 예제
                    /최종장: exercise, 과목당 1~2개의 증명(unknown problem)
                - dev(tool)
                    /실습: 코드스니펫화 
                    /최종장: 백엔드 프로젝트 2개(기본 CRUD ->실서비스 수준 API)
                - ml
                    /실습: 간단한 데이터 예제 
                    /최종장: 논문 reproduction
        2. 로그를 루틴화 시켰으나, 로그에 내재적인 구조가 부재 -> 공개적인 기록으로 커밋하기에 부실한(부끄러운) 글이 됨 
            => **템플릿 구축(커스텀 요구사항을 만족시키도록)**에 생성형ai의 도움을 받기
